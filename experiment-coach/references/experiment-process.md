# The Experiment Process

## The Hypothesis Formula

State the hypothesis specifically:
> "We believe [customer segment] has [need] and will [commit to action] because [reason]."

Then follow this process:

1. **Define the methodology** — How will you test this?
2. **Predict results** — What do you expect if the hypothesis is true?
3. **Run the experiment** — Time-box it with a deadline.
4. **Score results** — What was predicted vs. actual? What's the gap?
5. **Document learning** — What did you learn? What changes as a result?

## What makes a good experiment

- State the riskiest assumption upfront — what must be true for this to work?
- Define success/failure criteria before starting (not after)
- Find the fastest route to signal — don't build before you test
- Layer your effort — increase investment only after you've earned the right through learning
- Change one variable at a time

## The Validation Sequence

Follow this order. Don't skip steps:

1. **Validate the need** — Is there a real problem? This comes first.
2. **Map the business model** — Once need is validated, build the Lean Canvas.
3. **Identify your biggest risks** — What's still an idea, a hypothesis, or validated?
4. **Run experiments** — Test the highest-risk assumptions first.
5. **Build the playbook** — Document what works into a repeatable process.
6. **Scale** — Only now invest in scaling what's proven.

## The RISE Cycle

Research → Ideate → Set up → Evaluate. Each cycle generates evidence that either validates an approach (graduates to Project work) or invalidates it (narrows the search).

**Research:** Identify KPIs, analyse current performance, get user feedback, study competitors. Don't get stuck — wrap this up in a week or less.

**Ideate:** Start with high-impact, low-effort experiments. Prioritise with ICE + Pain scoring. Don't run experiments just because they're interesting — run them because they address the constraint.

**Set up:** Define the hypothesis, success/failure criteria, baseline, resources, timeline. One variable at a time.

**Evaluate:** Measure results against hypothesis. Document what you tested, what happened, what you learned, what you'll do next. This feeds directly into the next RISE cycle.

## Patterns from real experiments

Most GTM experiments fail because they assume success. The pattern that works: clear riskiest assumption, defined success/failure criteria, fastest route to signal. Quick wins include reframing conference strategy for learning (not leads), and validating channel traffic before investing further.

The layered experiment approach (test cheaply first, invest only after signal) consistently outperforms big-bang experiments. Example: testing ads-only before building landing pages and lead magnets.
